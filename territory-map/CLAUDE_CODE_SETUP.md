# MapKi — Claude Code Setup Guide

Full-stack franchise territory map app. React + TypeScript + Convex + Tailwind + Leaflet.

---

## Tech Stack
- **Frontend**: React 18 + Vite + TypeScript
- **Styling**: Tailwind CSS v4 + shadcn/ui (dark theme, slate-950 background)
- **Maps**: Leaflet.js (CARTO dark tiles — no API key needed)
- **Backend**: Convex (real-time database + auth) — 18 tables
- **Package manager**: Bun (preferred) or npm

---

## Initial Setup

### 1. Extract & install
```bash
tar -xzf territory-map-export.tar.gz
cd territory-map
bun install
# or: npm install
```

### 2. Connect your Convex deployment
You already have a Convex.dev account. Run:
```bash
npx convex dev
```
This will:
- Ask you to log in (if not already)
- Let you create a new project (e.g. "mapki")
- Auto-generate `convex/_generated/` files
- Set `VITE_CONVEX_URL` in your `.env.local`

### 3. Run locally
```bash
bun run dev
# App at http://localhost:5173
# Convex dashboard at https://dashboard.convex.dev
```

---

## Environment Variables

Create `.env.local` in the project root:
```
VITE_CONVEX_URL=https://your-deployment.convex.cloud
```
(Convex CLI sets this automatically when you run `npx convex dev`)

## Deploy to Vercel

### Option A — Vercel CLI
```bash
npm i -g vercel
vercel
# Follow prompts — it auto-detects Vite
```

### Option B — Vercel Dashboard
1. Push to GitHub first
2. Import repo at vercel.com
3. Add env var: `VITE_CONVEX_URL` = your Convex deployment URL
4. Deploy

## Database / Convex Schema

18 tables: `brands`, `territories`, `userProfiles`, `invites`, `crmLeads`, `brandNotifications`, `franchiseProfiles`, `prospectProfiles`, `categories`, `leads`, `savedItems`, `alerts`, `brandClaims`, `activityLog`, `contactTags`, `appNotifications`, `notificationDismissals` + auth tables.

See `convex/schema.ts` for full definitions.

To seed test data after connecting Convex:
```bash
# In Convex dashboard → Functions → seed:seedBrands
# Or via CLI:
npx convex run seed:seedBrands
```

## Important Notes for Claude Code

1. **Always keep `defaultTheme="dark"` in App.tsx** — changing it to "light" breaks button visibility
2. **Never import `{ Map }` from lucide-react** — use `{ Map as MapIcon }` to avoid shadowing JS global
3. **Bun lockfile**: If you change `package.json`, regenerate `bun.lock` with `bun install`
4. **Convex _generated**: Never edit files in `convex/_generated/` — they're auto-generated by the Convex CLI
5. **Logo**: Use `public/logo-dark-bg.svg` everywhere — the PNG breaks at small sizes
